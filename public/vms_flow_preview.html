🪪 ขั้นตอนการขออนุญาตและ Integration ThaiD

Phase 1 — ขออนุญาตใช้งาน ThaiD
  • 1หน่วยงาน (กระทรวงฯ) จัดทำ หนังสือราชการ ขอใช้บริการ ThaiD สำหรับระบบ VMS
  • 2ยื่นหนังสือต่อ กรมการปกครอง (DOPA) หรือ ETDA ตามแนวทางที่กำหนด
  • 3แนบเอกสาร: วัตถุประสงค์การใช้งาน, ขอบเขตข้อมูลที่ต้องการ, มาตรการรักษาความปลอดภัย
  • 4รอผลการพิจารณาและอนุมัติจากหน่วยงานต้นสังกัด
  • ✓ได้รับ Client ID + Client Secret สำหรับเรียก ThaiD API
Phase 2 — การเชื่อมต่อและทดสอบ
  • 1ทีม VMS Config Client ID/Secret ใน Backend สำหรับ ThaiD OAuth2
  • 2ทดสอบ OAuth2 Flow: Redirect → ThaiD App → Callback → รับ Token
  • 3ทดสอบดึงข้อมูล: ชื่อ-นามสกุล, เลขบัตร, วันเกิด จาก ThaiD API
  • 4ทดสอบกับอุปกรณ์จริงที่ Kiosk: ผู้ใช้เปิด ThaiD App → Scan QR → ยืนยัน
  • 5ทดสอบ Error cases: ปฏิเสธ, Timeout, ข้อมูลไม่ตรง
  • ✓ผ่านการทดสอบ → นำเข้าสู่ Production
